Учебная теория

  • Boot chain Boot ROM запускает SPL/MLO, затем полный U-Boot, затем kernel, final Device Tree, initrd и rootfs.
  • U-Boot Загружает kernel, DTB, DTBO overlays, initrd и bootargs. Root filesystem целиком в RAM не загружает.
  • Device Tree Паспорт железа для Linux kernel: адреса, IRQ, clocks, pinmux, status, compatible.
  • Driver matching Kernel сопоставляет compatible из DT с of_match_table драйвера и вызывает probe().
  • DT property и поведение Свойство Device Tree меняет поведение только если драйвер реально его читает.
  • IIO и userspace ABI Драйвер ADC создает IIO device, userspace читает in_voltageN_raw и in_voltage_scale.
  • Host-target workflow VS Code на хосте, передача по SSH/SCP/rsync, проверка на BeagleBone через make/insmod/dmesg.

База команд

0. Базовые shell-команды
Показать текущий рабочий каталог
pwd

pwd нужен, чтобы не перепутать host project, target shell и U-Boot prompt.

Показать файлы, права и скрытые артефакты
ls -la

-l включает подробный формат, -a показывает скрытые файлы вроде .hello.o.cmd.

Создать каталог вместе с родительскими каталогами
mkdir -p modules/hello

-p означает parents: создать недостающие каталоги и не считать ошибкой уже существующий путь.

Показать строки 1..120 из файла
sed -n '1,120p' modules/hello/hello.c

-n отключает автоматическую печать всех строк, а '1,120p' печатает только строки с 1 по 120.

1. Linux host и подготовка microSD
Проверить наличие базовых утилит на Linux-хосте
which lsblk xzcat sha256sum dd wget minicom
Безопасно определить microSD по списку дисков
lsblk -p -o NAME,SIZE,TYPE,MODEL,MOUNTPOINTS
Проверить SHA256 скачанного образа
sha256sum -c images/am335x-debian-12.13-base-v6.12-armhf-2026-03-17-4gb.img.xz.sha256sum
Проверить, что .xz архив читается
xz -t images/am335x-debian-12.13-base-v6.12-armhf-2026-03-17-4gb.img.xz
Проверить настройки первого запуска на BOOT-разделе
sed -n '1,220p' /tmp/bbb-boot/sysconf.txt
2. UART и первая загрузка
Посмотреть последние kernel messages после подключения адаптера
dmesg | tail -n 40
Открыть UART console 115200 8N1
sudo minicom -D /dev/ttyUSB0 -b 115200

Не подключать VCC USB-UART адаптера к debug-разъему BeagleBone Black.

3. U-Boot
Эти команды выполняются в U-Boot prompt =>, а не в Linux shell.
Версия U-Boot и toolchain
version
RAM, relocation, MAC, FDT address
bdinfo
Список MMC устройств и разделов
mmc list
mmc dev 0
mmc part
Читать environment без сохранения изменений
printenv

Не выполнять saveenv без отдельного явного решения.

4. Linux runtime на BBB
Понять, что команды выполняются на BeagleBone
hostname
uname -r
whoami
pwd
Зафиксировать running kernel
uname -a
Проверить bootargs, rootfs и console
cat /proc/cmdline
Сопоставить mmcblk0/mmcblk1 и mount points
lsblk -o NAME,SIZE,FSTYPE,LABEL,PARTUUID,MOUNTPOINTS
Посмотреть последние сообщения ядра
dmesg | tail -n 20
5. Device Tree
Прочитать model из Device Tree
tr -d '\0' < /proc/device-tree/model; echo
Посмотреть верхние уровни итогового DT
find -L /proc/device-tree -maxdepth 2 -type d | head -n 80
Посмотреть marker-файлы примененных overlays
find -L /proc/device-tree/chosen/overlays -maxdepth 1 -type f -print
Просмотреть U-Boot/Linux boot configuration
sed -n '1,220p' /boot/uEnv.txt

Не изменять /boot/uEnv.txt без предварительного бэкапа.

6. ADC через IIO
Найти IIO devices и имя драйвера
for dev in /sys/bus/iio/devices/iio:device*; do
  echo "=== $dev ==="
  cat "$dev/name" 2>/dev/null
done
Прочитать все raw-коды ADC
adc_dev=/sys/bus/iio/devices/iio:device0
for f in "$adc_dev"/in_voltage*_raw; do
  echo -n "$(basename "$f"): "
  cat "$f"
done
Прочитать коэффициент пересчета raw -> mV
cat /sys/bus/iio/devices/iio:device0/in_voltage_scale
7. Host-target сеть, SSH и rsync
Проверить, применить или удалить временный route helper
sh scripts/bbb-route.sh check
sh scripts/bbb-route.sh apply
sh scripts/bbb-route.sh delete
Проверить, не уводит ли VPN маршрут к плате
ip route get 10.129.1.152
Добавить route до платы в VPN policy table 7113
sudo ip route replace 10.129.1.152/32 dev eno0 src 10.129.1.110 table 7113
Проверить SSH без интерактивной сессии
ssh andrey@10.129.1.152 'hostname; uname -r; whoami'
Войти на BeagleBone как в обычный shell
ssh andrey@10.129.1.152

После входа prompt должен стать похожим на andrey@BeagleBone:~$.

Передать тестовый файл с хоста на плату
printf 'bbb host to target transfer test\n' > /tmp/bbb_ssh_transfer_test.txt
scp /tmp/bbb_ssh_transfer_test.txt andrey@10.129.1.152:/home/andrey/
ssh andrey@10.129.1.152 'cat /home/andrey/bbb_ssh_transfer_test.txt'
8. Kernel modules: подготовка исходников на хосте
Создать структуру исходников первого модуля
mkdir -p modules/hello
Показать исходник hello.c на хосте
sed -n '1,120p' modules/hello/hello.c
Показать Makefile для Kbuild
sed -n '1,40p' modules/hello/Makefile

Ожидаем строку obj-m += hello.o.

Синхронизировать исходники перед сборкой
rsync -av modules/hello/ andrey@10.129.1.152:/home/andrey/bbb-course/modules/hello/

-a включает archive mode, -v показывает передаваемые файлы, завершающий / у источника передает содержимое каталога.

9. Kernel modules: target-среда и Kbuild
Проверить headers/build tree текущего ядра
ls -ld /lib/modules/$(uname -r)/build
readlink -f /lib/modules/$(uname -r)/build
Проверить make, gcc и ld на BBB
which make gcc ld
make --version | head -n 1
gcc --version | head -n 1
ld --version | head -n 1
Собрать внешний модуль через Kbuild
cd ~/bbb-course/modules/hello
make -C /lib/modules/$(uname -r)/build M=$PWD modules
Убедиться, что hello.ko создан
ls -la
10. Kernel modules: metadata, загрузка и выгрузка
Проверить, где находятся утилиты управления модулями
dpkg -l kmod
which insmod rmmod lsmod modinfo
ls -l /usr/sbin/modinfo /sbin/modinfo
Проверить metadata и vermagic модуля
/usr/sbin/modinfo hello.ko
Загрузить hello.ko в ядро
sudo /usr/sbin/insmod hello.ko
dmesg | tail -n 20
lsmod | grep hello
Выгрузить модуль и проверить exit log
sudo /usr/sbin/rmmod hello
dmesg | tail -n 20
lsmod | grep hello
Альтернативный просмотр kernel log
journalctl -k -n 40

Рабочий цикл разработки

1. Редактирование

Пишем код и документацию в VS Code на Linux-хосте.

2. Передача

Передаем файлы на BeagleBone через SSH/SCP/rsync по Ethernet.

3. Проверка

На плате выполняем make, insmod, dmesg, rmmod и runtime checks.

Следующие шаги

  • Hello module completed Первый out-of-tree модуль собран на BBB, загружен через insmod, проверен через dmesg/lsmod и выгружен через rmmod.
  • Добавить параметр модуля Следующий учебный шаг: module_param, modinfo, загрузка с параметром и проверка через /sys/module.
  • Расширить debugging workflow После параметров перейти к pr_debug, dynamic debug и более аккуратной диагностике.
