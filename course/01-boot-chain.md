# Обзор цепочки загрузки

BeagleBone Black построен на TI AM335x. Загрузка - это цепочка небольших
программ, где каждый этап подготавливает достаточно железа для следующего.

## Этап 0: питание и reset

Питание подается через mini-USB или 5 V barrel jack. PMIC поднимает линии
питания, reset отпускается, и CPU AM335x начинает выполнять неизменяемый код из
внутреннего Boot ROM.

Контрольная точка:

- PWR LED горит постоянно.
- Через UART на этом этапе может еще ничего не выводиться: ROM обычно не печатает
  привычные boot-логи.

## Этап 1: AM335x Boot ROM

Boot ROM зашит внутри SoC. Он читает конфигурационные пины загрузки
(`SYSBOOT`) и пробует загрузочные устройства в заданном порядке. На BeagleBone
Black обычная загрузка чаще всего идет сначала с eMMC. Если зажать кнопку
`S2` / `BOOT` во время подачи питания, microSD становится предпочтительным
источником загрузки.

ROM загружает первый загрузчик во внутреннюю SRAM. Исторически этот файл часто
называется `MLO`; в современных терминах U-Boot это U-Boot SPL.

Что умеет ROM:

- Выбрать источник загрузки: eMMC, microSD, UART, USB и другие варианты,
  зависящие от пинов.
- Прочитать небольшой boot image с выбранного носителя.
- Запустить первый загрузчик.

Чего ROM не умеет:

- Монтировать полноценную Linux root filesystem.
- Инициализировать все периферийные устройства платы.
- Напрямую загрузить большое ядро из обычной структуры разделов.

## Этап 2: U-Boot SPL / MLO

SPL означает Secondary Program Loader. Он достаточно маленький, чтобы поместиться
в SRAM. Главная задача SPL - инициализировать DDR3 RAM и загрузить полноценный
U-Boot.

Обычно SPL делает следующее:

- Инициализирует тактирование и минимальное состояние пинов.
- Инициализирует DDR3 память.
- Читает полный U-Boot с microSD или eMMC.
- Передает управление полному U-Boot, который уже работает из RAM.

Контрольная точка:

- UART начинает показывать `U-Boot SPL ...`.
- Если загрузка останавливается здесь, частые причины: питание, разметка SD,
  неверный образ или ошибка инициализации DDR/storage.

## Этап 3: полный U-Boot

U-Boot - основной загрузчик. Он дает интерактивную консоль, переменные
окружения, boot-скрипты и возможность загрузить Linux.

Обычно U-Boot делает следующее:

- Определяет модель платы из EEPROM.
- Выбирает boot device.
- Читает `/boot/uEnv.txt`, extlinux config или distro boot scripts.
- Загружает kernel, initramfs и Device Tree Blob.
- Применяет Device Tree overlays для HDMI, eMMC, ADC, PRU, capes и pinmux.
- Передает kernel command line.
- Передает управление ядру Linux.

Контрольная точка:

- UART показывает `U-Boot ...`.
- Останавливаем autoboot нажатием любой клавиши.
- Полезные команды:

```text
version
help
bdinfo
mmc list
mmc dev 0
mmc part
printenv
ls mmc 0:1 /
ls mmc 0:3 /
ls mmc 0:3 /boot
boot
```

## Этап 4: ядро Linux

Ядро стартует с описанием железа из Device Tree. Оно инициализирует драйверы,
управление памятью, планировщик, прерывания, storage и networking.

Обычно ядро делает следующее:

- Разбирает Device Tree.
- Инициализирует периферию AM335x.
- Монтирует initramfs, если он используется.
- Находит и монтирует настоящую root filesystem.
- Запускает PID 1.

Контрольная точка:

- UART показывает строки Linux kernel log.
- Полезные команды после входа:

```sh
uname -a
cat /proc/cmdline
dmesg -T | less
ls /proc/device-tree
tr -d '\0' < /proc/device-tree/model; echo
```

## Этап 5: userspace

PID 1 обычно `systemd`. Он запускает сервисы: udev, сеть, SSH, USB gadget
networking, logging и login prompts.

Контрольная точка:

```sh
systemctl status
systemctl --failed
ip addr
journalctl -b
```

## Модель в голове

Вся цепочка:

```text
Power/Reset
  -> AM335x Boot ROM
  -> U-Boot SPL / MLO
  -> full U-Boot
  -> Linux kernel + Device Tree + initramfs
  -> root filesystem
  -> systemd
  -> login shell
```

## Что где находится и что можно менять

Важно разделять заводской код внутри процессора и файлы, которые лежат на
носителе загрузки.

```text
Boot ROM  - внутри AM335x SoC, изменить нельзя
SPL / MLO - на microSD или eMMC, можно заменить
U-Boot    - на microSD или eMMC, можно заменить и настроить
Linux     - на разделах образа, можно заменить и настроить
rootfs    - корневая файловая система Debian
userspace - systemd, сервисы, login, SSH и пользовательские программы
```

### Boot ROM

Boot ROM - это неизменяемый код, записанный производителем в AM335x. Он не
лежит на microSD или eMMC и не редактируется обычными средствами.

Его задача минимальная:

- прочитать состояние boot-пинов `SYSBOOT`;
- выбрать источник загрузки;
- найти первый маленький загрузчик;
- загрузить его во внутреннюю SRAM процессора;
- передать ему управление.

На BeagleBone Black кнопка `S2` / `BOOT` влияет на выбор источника загрузки.
Если зажать ее при подаче питания, microSD получает приоритет перед eMMC.

### SPL / MLO

SPL означает Secondary Program Loader. Для AM335x историческое имя первого
загрузчика - `MLO`. В современных сборках U-Boot это по сути U-Boot SPL.

SPL / MLO уже находится на носителе: microSD или eMMC. Его можно заменить,
перезаписав загрузочный носитель.

Почему нужен отдельный SPL:

- Boot ROM умеет загрузить только небольшой первый этап.
- На ранней стадии еще не инициализирована DDR3 RAM.
- SPL помещается во внутреннюю SRAM.
- SPL инициализирует DDR3 и загружает полный U-Boot в RAM.

### Full U-Boot

Полный U-Boot - это основной загрузчик. Он уже работает из RAM и умеет гораздо
больше, чем Boot ROM и SPL:

- выводить интерактивную консоль по UART;
- читать переменные окружения;
- работать с `mmc`, FAT/ext-разделами и файлами;
- выбирать kernel и Device Tree;
- применять boot scripts и overlays;
- передавать управление Linux kernel.

U-Boot можно заменить вместе с образом на microSD/eMMC, а часть его поведения
можно менять через environment и boot configuration.

### Linux и sysconf.txt

`sysconf.txt` не является загрузчиком и не участвует в запуске Boot ROM, SPL или
U-Boot. Это файл первичной настройки userspace.

Он лежит на FAT-разделе `BOOT`, чтобы его можно было легко открыть с Linux,
Windows или macOS до первого запуска платы. После старта Linux специальный
сервис читает `sysconf.txt` и применяет начальные настройки, например
`user_name=` и `user_password=`.

То есть место `sysconf.txt` в цепочке позднее:

```text
Boot ROM -> SPL / MLO -> U-Boot -> Linux kernel -> rootfs -> systemd -> sysconf
```

Вывод: Boot ROM выбирает и запускает SPL. SPL запускает полный U-Boot. U-Boot
запускает Linux. `sysconf.txt` применяется уже после запуска Linux userspace.
